Паттерн Адаптер используется, когда у нас есть старый класс с несовместимым интерфейсом (Adaptee), но клиентский код ожидает другой интерфейс (Target). Мы создаем класс-Adapter, который реализует целевой интерфейс, внутри содержит объект Adaptee и преобразует вызовы из одного формата в другой. В примере: клиент (NotificationService) хочет отправлять сообщения через единый интерфейс IMessageSender с методом Send(header, body). Но у нас есть старые сервисы: LegacyEmailService.SendEmail(fullMessage) и LegacySmsService.SendSms(phone, text). Мы создаем EmailAdapter и SmsAdapter, которые реализуют IMessageSender, внутри вызывают методы старых сервисов, преобразуя параметры (header+body склеиваются в письмо или header используется как номер телефона). В итоге клиент ничего не знает о старых классах и работает только через удобный ему интерфейс.
